# Slay the Spire 2 Card Scraper

This directory contains the context and structure of the scraped Slay the Spire 2 card data from `sts2.untapped.gg`.

## Architecture & Data Flow

All scraping and mass-downloading logic has been consolidated into a single unified script: **`sync_cards.py`**.

1. **`cards.json`**  
   This file acts as the primary source of truth for the card list. It is an exported JSON array automatically generated by `sync_cards.py` if it does not already exist in the directory.  
   Each entry is an object representing a unique card page:
   ```json
   {
       "link": "https://sts2.untapped.gg/en/cards/abrasive",
       "card_name": "abrasive"
   }
   ```
   *Note: The `card_name` is exactly the URL slug for the card.*

2. **`card_images_full/`**  
   This directory stores the actual high-resolution images generated natively by the untapped.gg frontend. `sync_cards.py` dynamically navigates the DOM using Playwright to extract the image components.
   
   The images directly map to the `cards.json` entries using the `card_name` value:
   
   - **Base Card:** Maps to `{card_name}.png` (e.g., `abrasive.png`)
   - **Upgraded Card:** Maps to `{card_name}-upgraded.png` (e.g., `abrasive-upgraded.png`)

### Edge Cases
- Some cards, such as "Ascender's Bane", do not have an upgraded form. For those cards, only the base `{card_name}.png` exists. `sync_cards.py` correctly identifies the absence of the "Upgraded" toggle UI and safely skips it without failing.

## Usage
Simply run the sync script to automatically resume downloading where it left off, or to backfill missing cards:
```bash
python3 sync_cards.py
```

If you need to iterate, display, or load the images programmatically in your own app, load the `cards.json` file first to retrieve the definitive list of cards. From there, you can dynamically construct the paths to the generated images in the `card_images_full` folder using the `{card_name}` convention.
